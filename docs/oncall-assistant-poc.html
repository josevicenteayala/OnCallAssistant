How it works

Capture · Ask · Answer.

1

Capture — every thread becomes knowledge

Slack message.channels events stream to the Ingest Lambda. All replies on a thread are merged into one document keyed by thread_ts, written to Amazon S3, and indexed by Bedrock KB.

2

Ask — engineer @mentions the bot

An engineer mentions @oncall-assistant with a free-form question — e.g. "what's happening with apt-service?". The Query Lambda sends it straight to the Knowledge Base. No thread lookup, no extra Slack calls.

3

Answer — top-5 past incidents, grounded

Bedrock KB runs cosine search over Amazon S3 Vectors, passes the top-5 retrieved chunks to Amazon Nova Lite under a strict retrieved-only · ≤ 5 bullets prompt, and posts the reply back in the same thread.

Architecture & flows

API GW → Two Lambdas. One knowledge base.

Ingest path

POST /events route · Idempotent on thread_ts · start_ingestion_job triggers KB sync · KB chunks then embeds via Titan V2 · resolution replies re-ingest automatically.

Query path

POST /query route · Stateless · retrieve_and_generate runs KB retrieval + Nova Lite prompt in one call · chat.postMessage posts in-thread · ≤ 5 grounded bullets.

Metrics & cost

Low-cost by design — serverless, pay-per-use.

Vector store

Amazon S3 Vectors

  • Storage · $0.06 / GB-month
  • Data upload · $0.20 / GB (PUT) · min 128 KB/PUT
  • Query request · $2.50 / million queries
  • Query data processed · $0.004/TB (≤100K vecs) · $0.002/TB (100K–10M) · $0.0004/TB (10M+)
  • Query data returned · $0.01 / GB · first 512 KB/query free

Conclusion

What the PoC proves.

What this proves

  • End-to-end loop works: Slack → index → @mention → grounded in-thread answer
  • Tribal Slack knowledge becomes semantically searchable with no manual curation
  • Continuous learning is free: resolution replies re-ingest automatically
  • Fully managed AWS stack — no servers, no vector DB to operate
  • Covers the real fault domains: Kubernetes, Kafka, IAM, TLS, business-logic incidents
Bottom line — a working, low-cost RAG agent that already shortens time-to-context during incidents. Path to production is well-defined; the remaining work is hardening, not invention.
PoC chose Amazon S3 Vectors over OpenSearch Serverless on cost; revisit at scale.
